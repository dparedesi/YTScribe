summary: This session, "Moody's: Architecting a Multi-Agent System on AWS," features Samuel Baruffi (Principal Solutions Architect at AWS) and Dennis Clement (Managing Director of Engineering and Architecture for Moody's Digital Content and Innovation), detailing how Moody's, a 100-year-old financial institution, revolutionized their customer service through an agentic AI system. Dennis explains Moody's evolution from a simple RAG application (Research Assistant, December 2023) to a sophisticated multi-agent orchestration system (end of 2025), driven by the critical requirement that 99% accuracy is insufficient for financial decisions involving billion-dollar loan origination, portfolio allocation, and regulatory compliance. The fundamental challenge was serving diverse customers (commercial banks, asset managers, insurance companies) needing expertise across multiple domains (credit ratings, climate risk, economics, compliance) while handling complex unstructured data, particularly financial PDFs with tables, charts, and footnotes spanning hundreds of pages. The architecture is built on five pillars: serverless infrastructure to handle spiky workloads (50X traffic increases during credit events), tools implemented as single-purpose Lambda functions for scalability and statelessness, two types of agents (simple agents with JSON-defined system prompts and curated tools orchestrated by the system, and complex agents as custom-built software running on ECS for long-running stateful tasks), a custom orchestrator managing 80+ tools and 100+ workflows (some exceeding 400 steps) with parallelization, cost optimization, and error handling, and model flexibility allowing each agent and tool to choose specific LLMs based on needs. The session highlights the critical challenge of processing unstructured financial documents, detailing four failed approaches (basic Python libraries losing context, custom parsing algorithms unable to scale, multi-modal foundational models too costly and inaccurate, and million-token context windows degrading performance and being expensive) before arriving at a multi-modal pipeline solution. This intelligent routing system classifies pages by content type: text-heavy pages go to Bedrock LLMs for OCR and markdown conversion, table-dominated pages use Bedrock Data Automation (BDA) for complex table extraction, and charts/images use vision models to create queryable metadata. The retrieval challenge is addressed through agentic retrieval, which decomposes complex queries into search strategies, executes multiple searches, reflects on completeness, and provides properly cited chunks, enabling intelligent document navigation rather than simple keyword or semantic search. Samuel covers AWS foundational services including Bedrock's global and geographic cross-region inference for capacity scaling, Knowledge Base for managed RAG workflows, and Bedrock Data Automation for extracting insights from complex documents. The system processes over a million tokens daily in production, delivering 60% faster insights and 30% reduction in task completions. The session concludes by discussing Agent Core primitives (Runtime, Gateway, Identity, Observability, Memory) that Moody's can leverage to reduce custom code and technical debt, and the future direction of exposing tools and workflows to customers through MCPs and smart APIs, enabling customers to build with Moody's intelligence rather than just consume it.
