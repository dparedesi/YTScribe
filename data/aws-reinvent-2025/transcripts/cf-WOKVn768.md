summary: This session demonstrates how Amazon developers leverage AI coding assistants to achieve significant productivity gains, showcasing proven prompt engineering techniques and the innovative Agent Standard Operating Procedures (SOPs) framework that transforms how teams interact with large language models. The presentation begins with fundamental prompt engineering strategies including zero-shot prompting for simple tasks, one-shot and few-shot prompting for complex tasks requiring specific patterns, system prompts providing contextual information about organizational structure and requirements, context prompts specifying how the LLM should think about problems, and role prompting where the LLM adopts specific personas like AWS senior solutions architect. Best practices emphasize combining these techniques for complex tasks by starting with context, defining the role, and specifying the task with desired output format. Advanced techniques include self-consistency prompting which generates multiple independent solutions to the same problem and asks the LLM to select the best option, improving reasoning quality and consistency for complex architecture decisions, troubleshooting, security analysis, and performance optimization. Tree of thought prompting provides explicit structured directions for problem-solving by defining multiple branches representing different aspects or approaches, with each branch explored systematically before arriving at recommendations, particularly effective for complex problems with multiple variables and decision-making scenarios involving numerous interactions. The explore-plan-code-commit workflow leverages LLMs' ability to understand codebases by first exploring existing code to understand structure and dependencies, then planning feature implementation based on codebase context, generating code that matches existing style and patterns, and finally committing with conventional commit messages following best practices. The session introduces Agent SOPs, a recently open-sourced framework that addresses the problem of developers copying and pasting magic prompt incantations by providing a standardized markdown format for creating reusable, structured prompts. Agent SOPs consist of a title, overview, parameter section with required and optional parameters, steps with RFC 2119-style constraints using keywords like MUST, SHOULD, MUST NOT, and MAY to precisely control behavior, examples for better LLM understanding, and troubleshooting steps. The format dramatically improves consistency compared to ad-hoc prompting, with agents following instructions much more reliably. The codebase summary SOP demonstrates this by analyzing codebases, generating comprehensive documentation including architecture diagrams using Mermaid, technology stack information, project structure, and consolidated agent-ready documentation files. The presentation showcases real-world implementation workflows including PDD (Prompt-Driven Development), an internal predecessor to spec-driven development that guides developers through requirements clarification, research, design, and implementation planning phases. The CodeTask Generator SOP creates detailed task descriptions similar to well-written issue tracker entries with background, technical requirements, dependencies, implementation approach, and acceptance criteria. The CodeAssist SOP implements test-driven development best practices through red-green-refactor phases, writing tests first and verifying they fail, implementing functional logic until tests pass, running all tests to ensure nothing breaks, refactoring to match existing code style, and committing with conventional commit messages. The session emphasizes that while AI tools excel at reducing the time spent exploring codebases and understanding context, there are limits to how well AI systems can handle extremely complex codebases with massive cognitive load requirements, using the analogy of a plumber who spends three days understanding a building before tightening one bolt. The presentation highlights that Agent SOPs are tool-agnostic and work with any LLM that can process the format, with thousands of SOPs used internally at Amazon for automating operations, coding, and productivity tasks including meeting transcription, task breakdown, and document management. Security considerations are addressed through Amazon Bedrock's account-level privacy guarantees, though best practices still recommend avoiding credentials and sensitive information in source code. The session concludes by demonstrating live implementation of a GitHub issue triage feature using these techniques, showing how developers can go from concept to production-ready code in days rather than weeks while maintaining code quality and following best practices.
