summary: This analytics leadership talk by Mai-Lan Tomsen Bukovec, VP of AWS Technology, explores three emerging trends in analytics: Agentic AI, Open Architectures, and Vectors. The first trend, Agentic AI, is rapidly integrating into AWS analytic services to automate data journeys, from code writing to data processing. AWS has launched an optimized Spark 3.5.6 engine across EMR, Glue, Athena Spark, and SageMaker notebooks, significantly boosting performance on Iceberg data. To facilitate upgrades, an AI assistance agent, trained on thousands of Spark upgrades, helps customers migrate applications in weeks instead of months, with FINRA anchoring their Spark version management on it. AI is also integrated into the new SageMaker notebook for data querying, exploratory analysis, and model development, allowing users to describe objectives in natural language and generating code, pipelines, and visualizations without managing infrastructure. The second trend, Open Architectures, is driven by the need for flexible, cost-effective analytic solutions. Open data formats like Apache Iceberg are key, enabling customers to swap analytic engines (Redshift, Athena, EMR, MSK, MSF, Snowflake) without expensive data migrations. AWS is introducing "analytics building blocks," disaggregated core primitives usable across services, such as a fully managed materialized view for Iceberg, which automatically detects changes and updates, appearing as a Glue Data Catalog table queryable by various tools. S3 tables, supporting Iceberg, also offer cross-region/cross-account replication and intelligent tiering. Tristan Baker from Intuit shares their data strategy for an AI-driven expert platform (TurboTax, Credit Karma, QuickBooks, MailChimp) serving 86 million consumers and 10 million businesses, managing 180 petabytes of data. Their strategy involves standardizing data semantics (common semantic layer with domains/subdomains), standardizing common data infrastructure (one-click tooling for app/service development, streaming/batch processing, BI systems, customer data cloud, Gen OS), enriching data with metadata (especially ownership, data products, semantic meaning), and establishing clear rules and requirements for data product quality. This has reduced "time to discover and access data" from 20 days to 9.4 hours and increased data cleanliness to 80%. A demo illustrates how Intuit Assist (Gen AI analytics assistant) uses rich metadata for semantic search and query generation, enabling seamless integration between TurboTax and Credit Karma. The third trend, Vectors, gives semantic meaning to data and extends agent memory. S3 Vectors, now GA, offers cost-effective, scalable storage for billions of vectors (up to 20 trillion per bucket) with high throughput and low latency, integrating with Bedrock knowledge bases and OpenSearch. OpenSearch has added AI capabilities like agentic search, semantic enrichment, GPU acceleration (indexing vectors 10x faster at 25% cost), and auto-optimization for vector indexing, making it easier to build billion-scale vector databases. The session concludes by emphasizing AWS's commitment to helping customers build future-proof data foundations at the intersection of data and AI.
