Mothership's architecture employs three pillars consisting of a backend service app, developer dashboard, and game SDK, with hosting organized around network-isolated verticals or cells where smaller games share resources while larger titles like Gorilla Tag receive dedicated infrastructure. The platform implements an innovative monolith-deployed-as-microservice pattern where compute and database shard independently by feature area with all instances running identical code, enabling feature-isolated scaling while maintaining simple intra-process communication contracts and single-version deployments that simplify operations for the small team of one to three developers. Critical to reliability is the fully continuous integration and deployment pipeline using trunk-based GitOps where merged pull requests trigger GitHub Actions building Docker images pushed to ECR, with CodePipeline serializing changes across verticals one at a time and CodeDeploy executing blue-green canary deployments that run integration tests via Step Functions and monitor CloudWatch alarms for automatic rollback on metric violations.
