Their test case involved running Illumina DRAGEN (FPGA-accelerated commercial software) on both previous-generation F1 and latest-generation F2 instances with identical samples and reference files. This performance gain is primarily attributed to the F2.6X large instances having 50% more VCPUs (24 vs. 16) and a 200% increase in cores per chip, which is crucial as DRAGEN processes run on a single FPGA chip. They use AWS Batch for HPC orchestration of containerized jobs, known for its massive scalability (e.g., scaling to 2.2 million VCPUs in 4 hours). Nextflow, an open-source workflow orchestrator, is used to define and submit pipelines. Data is staged in S3, containers are pulled from registries, and EC2 instances are dynamically provisioned and terminated, ensuring pay-as-you-go pricing. The Saarra platform provides a web UI for monitoring.
