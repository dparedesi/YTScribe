summary: This session demonstrates how to integrate AI agents with legacy applications using Kiiro IDE and Amazon Bedrock Agent Core Gateway without modifying existing codebases. Ryan Peterson, worldwide tech leader for modernization at AWS, frames agentic AI as the next decade's application modernization paradigm, following serverless, containers, and cloud-native architectures. The presentation traces the evolution from 2017 Transformers enabling parallel text processing, through scaling laws providing predictable performance improvements, few-shot reasoning emerging from larger models, chain-of-thought prompting improving mathematical capabilities, tool use enabling real-world interaction, ReAct framework for reasoning and acting, to Model Context Protocol (MCP) standardization released by Anthropic nearly a year ago. The session addresses the prototype-to-production chasm where 40% of agentic AI projects are canceled due to performance issues, scalability challenges, security concerns, and governance violations when attempting to build everything from scratch. AWS's approach centers on four pillars: state-of-the-art science through first-party models like Nova and partner models, purpose-built infrastructure for agents, specialized reusable agents, and intuitive developer experiences. Amazon Bedrock Agent Core provides a managed runtime with secure serverless environments, session isolation, multi-modal workload support, and long-running agent capabilities, complemented by Agent Core Gateway for connecting agents to legacy applications, Agent Core Browser for browser-based automation, Agent Core Code Interpreter for isolated code execution, plus identity, memory, and observability services. Agent Core Gateway solves the challenge of connecting agents to existing enterprise APIs, databases, and knowledge bases by transforming REST APIs and Lambda functions into MCP-standard tools, providing security and authorization for both inbound gateway access and outbound API calls supporting existing authentication frameworks, and integrating semantic search to handle exponential tool growth that causes latency and unpredictable results. The architecture follows MCP protocol where agents act as MCP clients connecting to Agent Core Gateway's MCP endpoint, which interfaces with REST endpoints or AWS Lambda functions. Setup involves configuring the gateway with OpenAPI specifications, connecting to existing identity providers like Okta, Azure AD, or Amazon Cognito, setting up targets for REST APIs using existing authentication mechanisms, and enabling semantic search that uses request context to filter relevant tools from potentially thousands of available tools, improving accuracy, speed, and cost. Strands Agents SDK, released as open source, abstracts agent development complexity, reducing weeks or months of work to minutes with just a few lines of code, supporting multiple AI agent runtimes, model choice flexibility, and tight integration with Agent Core and Amazon Bedrock. Kiiro, the AI-powered IDE, introduces spec-driven development where specifications are generated conversationally, reviewed and validated, progress through design phases, and automate task execution. The live demonstration shows creating a minimal terminal-based AI agent using Strands SDK in Kiiro, connecting it to a Swagger Pet Store application deployed on ECS Fargate without any code changes, configuring Agent Core Gateway by uploading the OpenAPI specification, setting up Cognito authentication, and updating the agent code to use the MCP endpoint, resulting in an agent that can query pets by status, search by tags for kid-friendly pets, retrieve detailed pet information, and combine API data with LLM general knowledge to provide contextual responses about pet adoption recommendations.
