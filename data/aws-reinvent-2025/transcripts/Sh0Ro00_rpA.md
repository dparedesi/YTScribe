summary: This AWS re:Invent 2025 session explores Amazon Bedrock AgentCore Memory, a fully managed service designed to provide intelligent agents with persistent memory capabilities for delivering personalized and context-aware interactions. Presented by Mani Kanuja, Principal Generative AI Specialist Solutions Architect, alongside Jay, Product Manager for AgentCore Memory, and Imran Shah, Senior VP of Engineering at Experian, the session demonstrates how memory serves as a critical but often overlooked component of agentic AI applications, bridging the gap between basic conversational agents and truly intelligent systems that can retain and leverage user context across interactions. The presentation distinguishes between two fundamental types of memory: short-term memory, which maintains raw conversation history and interaction states for periods ranging from seven days to one year, functioning similarly to recent human memory where details are recalled word-for-word; and long-term memory, which automatically transforms raw interactions into structured, persistent insights such as user preferences, summaries, and semantic facts that persist indefinitely across multiple sessions. The service addresses significant implementation challenges that organizations face when building memory systems from scratch, including managing separate storage solutions for different memory types, implementing automatic consolidation pipelines to handle conflicting preferences, maintaining secure multi-tenant isolation, and integrating comprehensive observability for debugging and troubleshooting. AgentCore Memory offers five built-in memory strategies: summary for condensing conversations, user preferences for capturing individual choices, semantic for extracting factual information, override for custom prompt and model control, and self-managed for complete control over extraction processes through S3 bucket delivery and dedicated APIs. A live demonstration showcases a PowerPoint builder agent comparing basic functionality without memory against memory-enabled capabilities, illustrating how the memory-enabled agent learns and applies user preferences for styling, technical fonts, purple color themes, and content requirements like incorporating AI ethics discussions without requiring explicit instructions on every interaction. Experian's case study reveals their transformative journey from fragmented, product-specific memory implementations using both managed solutions like OpenAI threads and custom open-source frameworks to a unified memory architecture leveraging AgentCore Memory, resulting in simplified operations, faster time to value, improved cross-agent interoperability, and better compliance with regulations like GDPR and CCPA through purpose-built memory governance capabilities that allow users to selectively delete their memories and administrators to perform periodic purges. The session emphasizes practical applications beyond content creation, including coding assistants that remember developer preferences for minimal code changes and specific programming languages, and customer support agents that recall past issues to provide consistent resolutions and better understand recurring problems, ultimately demonstrating that effective memory implementation is essential not only for accuracy and relevance but also for delivering superior customer experiences by making agents truly understand and remember their users' contexts, preferences, and interaction histories.
