summary: This AWS re:Invent customer panel session provides practical guidance on scaling AI applications using AWS infrastructure, featuring insights from ARM, Genentech, and Fireworks. The session introduces a framework for scaling AI workloads through three dimensions: scaling up by increasing compute power and memory per GPU, scaling out by adding more GPUs and servers across ultra clusters, and scaling across the inference stack. Andrea Klein from ARM addresses the challenges of managing GPU farms at scale, emphasizing that problems emerge predictably around 1000 GPUs and become unavoidable between 1000 and 10,000 GPUs. Key recommendations include consolidating machines in single locations within one ultra cluster or networking spine, choosing workhorse machines over hyper-optimized instances to maintain consistent utilization, using Kubernetes with managed services like EKS for cluster management, implementing specialized schedulers for batch workloads with gang scheduling capabilities to prevent deadlocks, and establishing comprehensive observability systems to monitor capacity utilization, workload efficiency, and cost allocation. The importance of node pools, storage hierarchies leveraging built-in NVMe SSDs for training data, and budgeting for CPU-only capacity, storage, and networking is emphasized. Challenges encountered include hardware faults requiring node testing and eviction procedures, networking issues best resolved by avoiding cross-zone or cross-spine traffic, and model implementation stability requiring careful testing of CUDA, EFA kernels, and library versions. Through distributed checkpointing, optimized data loading, and fully sharded data parallel training, Genentech improved GPU utilization from mid-60% to mid-90% for LLM training jobs.
