summary: This session presents Cathay Pacific's transformation from DevOps to DevSecOps using Agentic AI, presented by Mark Orrell from AWS Professional Services, Naresh Sharma, and Tony Leong from Cathay. The journey began with critical challenges: 78% of vulnerabilities identified by scanning tools were false positives, leading to thousands of hours of wasted time, delayed innovation, and significant time-to-market impacts. Cathay operates at massive scale with 70,000 daily passengers, 1,300 applications, 2,500 microservices, and 1,000 developers across 7 offshore development centers, making security a central concern. The traditional waterfall security testing approach created a tug-of-war between business teams wanting rapid market launches and security teams identifying vulnerabilities late in the development cycle, where fixing vulnerabilities costs 18 times more after development and UAT completion, and 64 times more if identified at launch. The transformation required a fundamental shift-left approach, embedding security, compliance, and best practices continuously throughout the software development lifecycle rather than only at the beginning or end. Key changes included automating all testing and scanning within the robust CI/CD pipeline, implementing blocking mode as a guardrail to prevent deployments with high and critical vulnerabilities, and addressing cultural challenges where security was perceived as an obstacle to innovation. The solution involved three parallel swim lanes: security design patterns and anti-patterns, enhancing DevOps capabilities to be DevSecOps-oriented, and security test automation including SAST for source scanning, IAST for interactive testing, and SCA for third-party library scanning. A critical component was the Security Champions program with three levels: Level 1 champions embedded in application teams to drive shift-left mentality and early security fixes, Level 2 senior personnel providing verification and secondary decision-making to prevent false positive approvals, and Level 3 distinguished engineers focusing on forward-looking security trends and market developments. The program included classroom training with comprehension tests, hands-on labs using real applications with problems, and AWS support for skills transfer, resulting in 86 participants across 15 sessions and 58 certified security champions. Despite achieving DevSecOps transformation with earlier vulnerability detection, a new challenge emerged: excessive manual effort required for false positive review and suppression, with an average of 500 exemptions needing review monthly, consuming 15 main days per month. The solution was developing an Agentic AI system that allows developers to interact with AI agents through an application security posture portal to ask about security recommendations, existing application posture, and request false positive exemptions. The AI agents learn from a knowledge base including approved exemptions, false positive patterns, company policies, and anti-patterns, then interact with different security tools to perform actual suppression and configuration updates, reducing average false positive review time from 30 days to 13 days (60% improvement). The agentic AI system uses AWS Bedrock for LLM capabilities, integrates with ServiceNow for workflow management, and handles complex decision-making beyond simple yes/no determinations, considering factors like whether vulnerabilities have been reported before, known remediations, compensating controls, and providing general vulnerability education. Key lessons learned emphasize that culture change takes time and requires clear guidelines and ambitious targets, tools require customization and vendor collaboration for optimization, and automation should evolve from human-in-the-loop to AI-in-the-loop automation. The presentation concludes with future plans including expanding security champions with more advanced skills, implementing more security scanning throughout the development lifecycle, and enhancing observability with more insightful dashboards for actionable security posture visibility.
