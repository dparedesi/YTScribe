summary: This comprehensive session provides foundational knowledge of AWS networking, covering global infrastructure with 38 regions, 43 local zones, and 146 Direct Connect points of presence connected via AWS's secure backbone network. The presentation begins with IP addressing fundamentals, explaining public versus private IP addresses for both IPv4 and IPv6, with AWS supporting over 150 services with IPv6 in dual-stack or IPv6-only modes. Amazon VPC fundamentals are explored in detail, including VPC IP Address Manager (VPC IPAM) for automated IP address planning and tracking across multiple accounts and regions, subnet creation with support for IPv4-only, dual-stack, and IPv6-only configurations, route tables for traffic routing, and security mechanisms including stateless network ACLs at the subnet level and stateful security groups at the ENI or instance level. Internet connectivity options are thoroughly covered, including public subnets with internet gateways supporting both IPv4 and IPv6, private subnets using NAT gateways for IPv4 and egress-only internet gateways for IPv6, DNS64 for backward compatibility in IPv6-only subnets, and VPC block public access for restricting internet connectivity. The session explores various connectivity methods including VPC peering for same-region connectivity with new VPC encryption controls for auditing and enforcing encryption in transit, Transit Gateway for scalable multi-VPC connectivity with layer 3 routing, and AWS Cloud WAN for global network management with dynamic routing and segment-based isolation. Hybrid connectivity options include AWS Direct Connect for private dedicated connections with support for public VIFs, private VIFs via Direct Connect gateways, and integration with Transit Gateway and Cloud WAN, site-to-site VPN with support for virtual private gateways and Transit Gateway termination, and SD-WAN appliance integration. Application networking is covered through AWS Private Link for accessing AWS services and custom services with unidirectional connectivity and service permissions, VPC resource endpoints for accessing resources without network load balancers, and Amazon VPC Lattice for layer 7 service networking with service networks, associations, and endpoints supporting zero-trust principles. DNS capabilities include Route 53 resolver with DNS firewall for blocklists and allowlists, resolver query logging, private hosted zones for internal DNS, inbound and outbound resolver endpoints for hybrid DNS, and Route 53 profiles for managing DNS configurations at scale across multiple accounts. Traffic inspection architectures are detailed, including distributed models using AWS Gateway Load Balancer for third-party appliances or AWS Network Firewall for fully managed inspection, and centralized models using Transit Gateway or Cloud WAN with inspection VPCs or native firewall attachments for VPC-to-VPC, on-premises, and internet egress inspection. Remote access solutions include AWS Client VPN for network-based access with certificate authentication and authorization rules, and AWS Verified Access for application and resource-based access with trust providers, device information, and continuous authentication supporting HTTP/HTTPS applications and TCP resources. Observability and monitoring tools are covered including VPC Flow Logs for flow-level visibility, Network Manager for topology and routing updates, CloudWatch Internet Monitor for internet performance metrics, Network Synthetic Monitor for hybrid connectivity metrics, Network Flow Monitor for intra-AWS performance, VPC Reachability Analyzer for hop-by-hop troubleshooting, and Network Access Analyzer for identifying unintended network paths. Throughout the session, both presenters emphasize IPv6 support, security best practices, scalability considerations, and the importance of proper network architecture design for enterprise deployments.
