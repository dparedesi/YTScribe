summary: "This AWS re:Invent 2025 keynote, delivered by AWS Senior Vice President Peter DeSantis and Dave Brown, Vice President of Compute and Machine Learning Services, explores how the AI transformation is driving massive infrastructure innovation while maintaining the fundamental Cloud attributes that have defined AWS since its inception: security, availability, elasticity, cost optimization, and agility, with DeSantis emphasizing that these core principles are more important than ever as AI-driven applications create unprecedented demands on infrastructure with bad actors using AI to attack systems, requiring tested Cloud platforms that provide capacity planning-free elasticity even for expensive AI workloads and the breadth of building blocks necessary for rapid innovation and pivoting. DeSantis recounts a foundational AWS innovation story from 2010 when engineers tackled the \"jitter problem\" in EC2 virtualization that created performance outliers for demanding workloads, ultimately leading to the creation of the Nitro system, AWS's custom silicon that moved virtualization off servers onto dedicated hardware to eliminate jitter entirely and achieve better-than-bare-metal performance while improving security and enabling diverse instance types, a development significant enough that the seventh edition of the classic computer science textbook \"Computer Organization and Design\" by Patterson and Hennessy now includes a section covering the Nitro system and Graviton processors. Dave Brown announces the preview of Graviton5-based M9g instances featuring 192 cores in a single package with over five times the L3 cache of previous generations, providing each core up to 2.6 times more L3 cache than Graviton4 and delivering up to 25% better performance with the best price-performance in EC2, achieved through innovations including direct-to-silicon cooling that removes the protective lid and reduces thermal interface material layers to drop fan power by 33%, with early customer results showing Airbnb seeing 25% better performance, Atlassian experiencing 20% lower latency, and SAP achieving 60% better performance on OLTP queries for SAP Hana Cloud. Payam Mirrashidi, Vice President of Cloud Systems and Platforms at Apple, shares how Apple has seen 40% performance increases and 30% cost reductions by rewriting core services in Swift and running them on Graviton, with features like spam detection in Messages using Swift on Graviton to scale compute-heavy operations with homomorphic encryption for hundreds of millions of users, and announces the first native Swift toolchain for Amazon Linux, enabling developers to leverage Swift's performance, modern design, and safety features for server-side development. Brown introduces Lambda Managed Instances, which bridges serverless simplicity with infrastructure control by allowing Lambda functions to run on EC2 instances inside customer accounts while Lambda manages provisioning, patching, availability, and scaling, enabling workloads like video processing and ML pre-processing that historically lived outside Lambda without requiring rearchitecture or adding operational burden. DeSantis unveils Project Mantle, the new inference engine powering Amazon Bedrock models, which introduces service tiers (priority, standard, flexible) for request prioritization, per-customer queue isolation for fairness, and Journal integration for durable transaction logs that enable fault-tolerant long-running requests and efficient fine-tuning directly within Bedrock, while Peter announces that Trainium is becoming PyTorch native with a simple code change from CUDA to Neuron enabling seamless migration, and Dean Leitersdorf, CEO of Decart, demonstrates real-time visual intelligence running on Trainium3 with live video generation achieving four times better frames-per-second performance than state-of-the-art GPUs through 80% Tensor Core utilization, showcasing how Trainium3's second-generation UltraServer with 144 chips, 360 PetaFLOPS of compute, and custom Neuron switches delivers differentiated AI performance."
