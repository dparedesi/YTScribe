summary: This session, presented by Vishal (EC2 Mac Product Manager) alongside Miranda Pearson from Riot Games and Tony Subanen from Supercell, details how AWS EC2 Mac enables Apple platform development at scale by eliminating the need for physical Mac hardware in data centers. EC2 Mac was introduced in 2020 to address key customer pain points: infrastructure overhead requiring physical data center visits, inability to automate fleet management, and lack of scalability for variable workloads. The service uses Apple hardware (initially Intel Mac Minis, now M2, M2 Pro, M4, and M4 Pro) retrofitted in AWS data centers with PCIe-to-Thunderbolt connections integrated with AWS Nitro for networking and EBS bandwidth. Recent improvements include M4 and M4 Pro support with 2TB local NVMe storage, dedicated host auto-recovery, reboot-based migration, Amazon DCV support for 4K GUI access, System Integrity Protection (SIP) configuration automation, and Mac OS 26 AMIs. Miranda shares Riot Games' migration journey from 60 on-premises Macs in a Las Vegas data center plagued by hardware failures (drives melting, 2% annual loss rate, 8% fleet downtime per repair) to EC2 Mac, resulting in 3x faster main build times, 2x faster signing workflows, and 2.5x faster code builds. Their infrastructure uses Terraform for provisioning, centralized AMIs with security features, Jenkins-triggered builds, SSM for bootstrapping, and event-driven automation. Tony details Supercell's approach using virtualization on EC2 Mac with TART (based on Apple's native virtualization framework) to enable rapid switching between VM images, avoiding AMI scrubbing delays. Their architecture leverages auto-scaling groups with AWS License Manager for dedicated host provisioning, EFS for storing TART VM images, local NVMe for high-performance builds, and a custom "Coco agent" for routing between Jenkins and GitHub Actions. Supercell achieved 10x more build capacity with only 3x the number of Macs, scaling from 150 daily builds to over 1,800, while reducing operational overhead from 5-7 people to 1.5 people. Both customers emphasize the benefits of eliminating physical hardware maintenance, improving build reliability, enabling cost optimization through auto-scaling, and unlocking faster innovation cycles. Future enhancements include exploring shared capacity, migrating more workloads to M4/M4 Pro, asset caching optimizations, and dynamic auto-scaling based on queue sizes.
