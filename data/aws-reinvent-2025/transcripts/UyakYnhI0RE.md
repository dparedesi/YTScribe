GuardDuty is a threat detection service that monitors log sources like CloudTrail logs and VPC flow logs, applying machine learning and detection rules for real-time threat identification, while GuardDuty Runtime Monitoring is an optional solution that deploys an eBPF agent to AWS compute workloads such as EC2 instances to gather operating system level events for monitoring and threat detection. Wasiq emphasizes that testing runtime monitoring is crucial both for evaluating whether the service meets security requirements and for validating incident response capabilities, but many customers use simplistic tests that map to single MITRE techniques like creating cron jobs to test persistence detection, which fails to provide insights into the solution's noise reduction capabilities since such actions are common in benign activity. GuardDuty gathers real-world attack information primarily from two sources: the AWS Customer Incident Response Team that assists customers during security incidents and provides data showing the top two initial access techniques are compromised credentials and public-facing application compromise, and threat intelligence from in-house and third-party sources that reveals indicators of compromise, tactics and techniques, activity patterns, and critical context about targeted applications and attack sequences. Wasiq demonstrates these concepts through a typical web application compromise scenario captured by honeypots, where attackers exploit vulnerabilities to deploy webshells, execute commands to download shell script malware, download second-stage malware like crypto miners, make files executable with chmod, execute malware, and persist through scheduled tasks, with each step mapping to specific MITRE tactics and revealing patterns like using curl to pipe downloaded scripts from IP-based URLs directly to shell processes and writing cron instructions to temporary files before loading with crontab commands. To enable realistic testing, AWS has added multiple scenarios to the Amazon GuardDuty Tester public GitHub repository, which provides easy instructions to set up secure lockdown test environments in AWS accounts with VPCs, EC2 instances, ECS clusters, and EKS clusters.
