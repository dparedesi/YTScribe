summary: This AWS re:Invent lightning talk introduces Innovation Sandbox on AWS, a solution designed to address the critical challenge of enabling developers to upskill and innovate while maintaining cost control and management oversight in cloud environments. The presentation begins by highlighting the problem that heavily regulated companies face with service allow lists that create bottlenecks preventing developers from testing and learning new AWS services, often resulting in approved services that never get used because developers couldn't experiment with them in real-world scenarios. The primary goal of Innovation Sandbox is to democratize innovation and upskilling on AWS by providing a managed environment where developers can safely experiment. The solution addresses common pain points with traditional sandbox implementations, including cost overruns from multiple users sharing the same sandbox, management overhead from tracking and cleaning up resources, and the difficulty of maintaining isolated environments. Innovation Sandbox, which went generally available in May 2024, is deployed via CloudFormation templates (four templates requiring approximately one hour to deploy), is fully supported and maintained by AWS, with code available on GitHub. The solution operates through three distinct personas: administrators who deploy the solution and manage underlying AWS accounts, managers who don't need to be technical but can delegate sandbox accounts to end users while controlling costs and durations, and users who request and utilize sandbox accounts for experimentation. The workflow involves administrators pre-creating AWS accounts that are placed into an account pool, managers provisioning or leasing these accounts to users through lease templates that define cost and time controls, and users requesting access through a user interface integrated with IAM Identity Center (compatible with third-party providers like Okta and Entra ID). Lease templates enable managers to configure multiple thresholds and actions based on budget and duration, including sending alerts, freezing accounts (which revokes user access while maintaining manager and admin access), and cleaning up or recycling accounts. The solution supports three primary account lifecycle patterns: freezing accounts for temporary suspension (useful for hackathons where some participants want to continue working), ejecting accounts from the solution to move them to development environments when projects prove valuable, and recycling accounts through automated cleanup using AWS Nuke that removes all resources and returns accounts to the pool. The live demonstration showcases the user interface capabilities, including account pool management where administrators can add accounts by moving them to an entry organizational unit, view account statuses (available, active, frozen), manually eject accounts, and directly access accounts through IAM Identity Center authentication. The lease template configuration interface allows managers to set maximum budgets with escalating actions (alerts at $50, freezing at $75, wiping at $100), duration-based controls with similar action thresholds, optional approval workflows, visibility settings (public templates visible to users versus private templates only assignable by managers), and cost reporting features that assign metadata for cost center tracking and generate monthly reports to S3 buckets. The user interface provides different views for each persona, with users able to see their active leases, request new leases from public templates, accept configurable terms of service, and directly access their sandbox accounts, while managers can approve requests, assign leases directly (particularly useful for private templates with higher budgets), modify individual lease thresholds, and monitor all leases across the organization. The solution includes comprehensive API support with a full OpenAPI specification, enabling integration into existing workflows and processes. Common use cases span universities needing isolated learning environments, highly regulated customers requiring controlled experimentation spaces, hackathons requiring temporary accounts with automatic cleanup, and generative AI development where teams need secure, isolated environments with cost controls for testing agentic applications and building AI solutions. The solution emphasizes configurability through AppConfig in the hub account, allowing organizations to customize terms of service, maximum sandbox limits per user, absolute maximums for lease templates, and cost report group requirements, making it adaptable to diverse organizational needs while maintaining security and cost governance.
