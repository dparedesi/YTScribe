summary: This session demonstrates how to build agentic workflows on AWS using third-party agents and tools from AWS Marketplace, covering the complete journey from architectural foundations through hands-on implementation to production deployment of pre-built solutions. The presentation begins by establishing that AI agents require agency (ability to act), instruction-taking to form goals and objectives, and reasoning and planning capabilities to invoke appropriate tools, representing an evolution from static workflow bots through cognitive automation and generative AI assistance to fully autonomous agentic systems where human roles shift from doing work to oversight. AWS focuses on three core areas: choice (bringing latest market-leading capabilities allowing customers to freely innovate and realize ROI quickly), trust (controlling, governing, establishing guardrails and compliance for probabilistic AI systems through transparency via AI service cards, bedrock guardrails for model safety, and audit capabilities through agent core observability with CloudTrail and CloudWatch), and scale (enterprise-grade capabilities enabling rapid innovation, production deployment, and scaling as demand grows). Amazon Bedrock AgentCore provides the foundation with three key areas: tools and memory (code interpreter for secure sandbox code execution, browser tool for web navigation, session and long-term memory capabilities), secure deployment at scale (runtime for containerized code deployment within customer VPCs with isolation guarantees, identity services for user and agent access control), and observability (OpenTelemetry integration with CloudWatch and CloudTrail for inspecting agent runs, diagnosing root causes, and understanding execution patterns). AWS Marketplace addresses the critical challenge of tool access by providing an enterprise-ready digital catalog of over 2,300 AI agents, tools, and agentic applications that reduce procurement time from months to days or minutes, offer unified billing and cost control transparency through AWS billing integration, and enable simple deployment with native AWS service integration allowing single-click agent deployment on AgentCore runtime. Marketplace products fall into three categories: API-based remote tools (like Tavily search) that act as external targets imported into AgentCore Gateway with automated metadata pulling, API schema translation, and API key rotation; container-based agents designed specifically for AgentCore runtime running inside customer VPCs with full AgentCore service benefits; and agentic applications plus professional services. Workday's case study, presented by Meera Rajaram, demonstrates building a planning agent representing four personas (analyst, modeler, planner, admin) with data exploration skills allowing analysts to conduct natural language ad-hoc analysis on vast datasets. Their journey involved multiple iterations: first attempting to send all data directly to LLMs (failing due to token limits, high costs, and math accuracy issues), then chunking data (still expensive and inaccurate), finally landing on having LLMs write Python code for data analysis executed in a sandbox code interpreter, with the LLM explaining results in natural language. Workday evaluated building their own sandbox but chose AgentCore code interpreter for security (code runs in secured virtual computers with data wiped clean after sessions), stateful multi-step analysis capabilities, and scalability (handling 100MB files for inline processing or gigabytes via S3). The solution took 1 software developer, 2 engineering managers, and 3 days from POC to production, achieving 50% token reduction, significantly improved accuracy, eliminated token limit concerns, and added only 400-500ms latency (1-2 seconds at 90th percentile). The hands-on demonstration shows building a company research agent for sales teams using LangChain framework, Anthropic Claude 3.5 Sonnet from Bedrock, Tavily search tool via AgentCore Gateway, and local S3 document search, walking through architecture design, cloud environment setup (model selection, API key management in Secrets Manager, gateway tool registration), and Python code implementation showing agent executor orchestration, prompt engineering defining agent objectives and available tools, and tool function implementation with JSON-RPC packaging for gateway invocation. The live demo shows the agent receiving a query about AWS sustainability, reasoning to invoke the search web tool with a refined query, receiving search results with citations, and curating a comprehensive response. Production considerations include security and governance through AgentCore runtime containerization, bedrock guardrails implementation, performance optimization through caching, reliability with proper failover mechanisms, and cost control for LLM token usage and tool call expenses. The session concludes with deploying a pre-built CRM AI sales agent from Marketplace in minutes: discovering the agent through natural language search or requirements document upload, single-click subscription with terms acceptance, and deployment providing only environment variables (CRM credentials stored securely in Secrets Manager) with the agent running entirely within customer VPC boundaries, inheriting vendor testing and optimization while maintaining full customer control.
