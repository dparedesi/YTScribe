summary: This session explores AWS storage innovations, emphasizing the building blocks philosophy where services are designed to remove friction and enable builders to focus on higher-level work, with S3 now 19 years old and processing over 500 trillion objects and 200 million requests per second worldwide. The presentation highlights that 80% of storage work involves continuous refinement and polishing, with over 1000 launches since 2020, exemplified by features like conditional put operations that enable atomic state transitions and have been adopted for 2% of all S3 puts (tens of millions of requests per minute), allowing customers to delete complex external locking code. At the hardware level, AWS introduced metal volumes, a disaggregated storage architecture where Nitro compute nodes securely virtualize physical disks, decoupling compute from storage to improve availability, enable flexible compute sizing, and realize over 10% power savings, with early deployment already at 53 exabytes. S3 Express, the high-performance zonal S3 service, has achieved up to 85% price reductions, improved performance to 2 million requests per second, and serves customers like Meta with over 60 petabytes, a million transactions per second, and 140 terabits per second of traffic. The session announces the general availability of S3 Vectors, a vector database API that enables similarity search for retrieval-augmented generation and other applications, which during its 5-month preview saw over 250,000 vector indices created, 40 billion vectors ingested, and over a billion queries served. The implementation uniquely addresses the cost challenge where vector indices for text can be 2-3x larger than the source data by using S3's throughput-optimized architecture rather than expensive in-memory storage, encoding vector neighborhoods as objects that are promoted to faster memory during queries and searched in parallel, achieving query latencies down to 100 milliseconds with support for up to 2 billion vectors per index and 10,000 indices per bucket. Customer examples include GoPro managing 450+ petabytes and a billion media assets, using vectors to solve the "fire and forget" content discovery challenge where users need to find specific moments in long-form video, and BMW building a vector-integrated data lake with natural language search across both structured data (via Athena) and unstructured data (via S3 Vectors). S3 Tables, launched last year with over 400,000 tables now in production, provides managed Apache Iceberg tables on S3, with recent improvements including 90% compaction price reduction through optimization, cross-region replication with "don't leave without the kids" consistency ensuring atomic snapshot visibility, intelligent tiering that doesn't interfere with background maintenance tasks, and support for Iceberg V3 features like row lineage and deletion vectors. The session introduces S3 Metadata as a managed table providing both journal and inventory views of bucket contents, enabling SQL queries and natural language queries against bucket metadata, with extensions to CloudWatch Logs, Storage Lens, and SageMaker Unified Studio creating a system of AWS service tables that can be joined for powerful cross-service analysis like triangulating access denied errors across S3 access logs, VPC flow logs, and object metadata. Finally, the presentation covers enterprise migration support through FSX access points for S3, now extended to NetApp ONTAP in addition to OpenZFS, allowing customers to replicate enterprise filer data to the cloud and then access it via S3 APIs for cloud-native applications like Athena, Bedrock Knowledge Bases, Glue, and SageMaker, bridging the gap between enterprise storage constructs and cloud-first development.
