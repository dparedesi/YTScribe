summary: This presentation shares Anthropic's learnings from building AI agents throughout 2025, focusing on Claude Code development and enterprise agent deployment patterns. The journey began with Claude 2.1 featuring a 200,000 token context window, but the real transformation occurred with Claude 3 Opus becoming a frontier model, followed by Claude Sonnet 3.5 demonstrating strong coding capabilities through the artifacts feature. The speaker's personal experience building Claude Code on a Friday night, creating a note-taking app without writing code, illustrates the power of agentic systems. Anthropic's evolution shows rapid model improvements, with Opus 4.5 scoring 80% on SWE-bench (up from 49% with Sonnet 3.5V2 just one year prior) while using fewer tokens than previous models, demonstrating that more expensive models can actually reduce total costs by solving problems more efficiently. The presentation distinguishes between workflows (multiple LM calls chained together with deterministic logic) and agents (LLM running in a loop with tools), explaining that agents solve workflow limitations by not requiring every edge case to be coded in and being more robust to errors, with Claude able to recognize unexpected results and try alternative approaches. Context engineering emerges as the critical discipline, replacing simple prompt engineering with a broader approach encompassing system prompts, tool definitions, tool responses, memory management, and context window compression strategies. Key context engineering principles include finding the Goldilocks zone between too specific (overwhelming the model with 32-page SOPs) and too vague (insufficient instructions), starting vague and iteratively adding specificity based on testing, and focusing heavily on tool design as the most important aspect of agent architecture. The presentation emphasizes progressive disclosure through skills (hiding instructions, scripts, and templates until needed), avoiding upfront data retrieval in favor of letting agents discover information through tools, and handling long-horizon tasks through compaction (summarizing and restarting), memory systems (file-based knowledge storage as demonstrated in Claude Plays Pokemon), and sub-agent architectures for research delegation. Context window management is crucial for reliability, avoiding context rot (degradation at high token counts), and leveraging prompt caching for cost and latency benefits. The Cloud Agent SDK is introduced as Anthropic's solution for programmatic agent access, extracting the core primitives from Claude Code (agent loop, system prompts, tools, permissions, memory) into a reusable SDK that Anthropic now uses for all new agentic products. The presentation predicts that 2026 will be the year of giving agents access to computers, with many problems mappable to coding problems (like creating PowerPoints and spreadsheets through code execution rather than specialized tools), and memory solvable through file systems. The talk concludes with guidance on choosing agentic frameworks, emphasizing the need for appropriate control levels, avoiding over-opinionated scaffolding, and ensuring the ability to tune key system components, swap prompts, and implement multi-agent architectures.
