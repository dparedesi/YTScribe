The presentation begins by defining resilience as the ability of a workload to recover from disruption, dynamically acquire resources, and mitigate disruptions, with two pillars: availability (proportion of time a workload is available, measured historically as five 9s) and disaster recovery (techniques to recover workloads using RTO—recovery time objective—and RPO—recovery point objective, maximum data loss). For Aurora PostgreSQL and MySQL, the first question addresses database node failures: Aurora storage (Grover) provides multi-tenant storage across 3 availability zones, automatically creating 6 copies of every write across 3 AZs (customers pay for one copy), enabling loss of an entire AZ plus one more storage node without durability issues, achieving RPO of 0 with no asynchronous replication out of the box. The storage architecture uses log-structured design, writing only log records (not pages) with storage applying logs to create pages on demand, maintaining a complete timeline of database commits. Backup and restore features include continuous backup to S3 by the storage engine (not impacting head node performance), enabling point-in-time restore from up to 35 days ago to 5 minutes ago for busy databases. For availability, adding a standby replica enables failover within 30 seconds with RPO of 0 due to shared storage, achieving multi-AZ availability patterns. The second question addresses performance resilience: Aurora supports up to 15 read replicas for read scaling, offloading reads with up to 100 milliseconds stale data, isolating workloads like analytics on separate replicas with dedicated buffer pools, supporting instances up to 48Xlarge (1.5 terabytes memory each), with automatic storage scaling in both capacity and performance without provisioning. Instance scaling includes failover tiers determining failover order, and DB.servius elastic instances scaling from 0 to 256 Aurora capacity units. DSQL is active-active, meaning any connection can read or write any data at any time from any QP in any availability zone.
